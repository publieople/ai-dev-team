# Agent类型定义

## 主Agent (CEO)

**职责：**
- 项目初始分析
- 需求对齐与确认
- 任务拆分与规划
- 子Agent生命周期管理
- 状态机维护
- 验收决策
- 异常上报

**不直接执行：**
- 不写具体业务代码
- 不直接运行测试
- 不直接修改配置文件

**决策权限：**
- 创建/销毁子Agent
- 任务优先级调整
- 重试策略制定
- 上报人类判断

## 子Agent类型

### Developer (开发工程师)

**职责：**
- 功能实现
- Bug修复
- 代码重构
- 编写测试

**输入：** TaskCard + 代码上下文
**输出：** 代码变更 + 执行报告

**生命周期：**
```
创建 → 接收任务 → 读取上下文 → 执行 → 生成报告 → 销毁
```

### Tester (测试工程师)

**职责：**
- 编写测试用例
- 执行测试
- 生成测试报告

**输入：** 待测试功能 + 测试要求
**输出：** 测试结果 + 覆盖率报告

### Researcher (技术研究员)

**职责：**
- 技术调研
- 方案对比
- 文档获取

**输入：** 研究课题
**输出：** 研究报告 + 推荐方案

### DocWriter (文档工程师)

**职责：**
- 维护技术文档
- 编写API文档
- 更新README

**输入：** 代码变更 + 文档需求
**输出：** 文档更新

## Agent创建规范

**命名规则：**
```
{role}-{project}-{seq}
例如：dev-shop-001, test-shop-001
```

**创建时机：**
- 任务进入 ASSIGNED 状态时
- 每个任务创建独立Agent
- 不允许复用Agent

**销毁时机：**
- 任务完成（成功/失败）
- 超时
- 被主Agent强制终止

## Agent能力声明

每个Agent启动时声明能力：

```json
{
  "role": "developer",
  "capabilities": ["typescript", "react", "nodejs"],
  "limitations": ["no-ui-testing", "no-database-setup"]
}
```

主Agent根据能力匹配合适的Agent类型。
